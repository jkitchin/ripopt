# ripopt v0.8 — Ipopt alignment & heuristic retirement plan

Synthesized from six concurrent ipopt-expert reviews (one per subsystem) of the
current `v0.8/feral-default` branch against `docs/IPOPT_ALGORITHM_SPEC.md` and
the Ipopt 3.14 reference under `ref/Ipopt/`. The goal is to align ripopt with
the well-tested Ipopt code paths and retire ripopt-only heuristics that work on
easy problems but fail on harder, larger, sparser ones.

Ground rules from `CLAUDE.md` carry through: **principled changes**, one
testable hypothesis per change set, isolated commit-or-revert per set, narrow
test target before full regression, no problem-specific tuning.

---

## 1. Tier 0 — verified bugs (do these first; high confidence, low risk)

**STATUS: all 14 items landed on `v0.8/feral-default`.** 19 commits across 5
worktree-isolated agents (Convergence/Init/Filter/Restoration/KKT), merged
sequentially in commits `151e905`, `cee3c3b`, `56b57f0`, `96022f7`, `7a78ff9`.
333 tests pass (220 lib + 113 integration), 0 failures. Benchmarks not yet
run.

These are clear deviations from Ipopt with verified `ref/Ipopt/...` line
numbers. Each is independently fixable without touching the surrounding
algorithm.

### T0.1  bound_push formula missing `max(|x_L|, 1)` magnitude scaling
- **Where:** `src/ipm.rs:9446-9462` (`push_initial_point_from_bounds`).
- **Spec:** Ipopt `IpDefaultIterateInitializer.cpp:544-560`:
  `pL = min(κ1·max(|x_L|, 1), κ2·(x_U−x_L))` (and symmetric for upper).
- **Bug:** ripopt uses `min(bound_push, bound_frac·range)` with no magnitude
  scaling. For one-sided bounds at large magnitude (`x ≥ 1e3`), ripopt pushes
  by `0.01` absolute when Ipopt pushes by `10`.
- **Test hypothesis:** problems with large-magnitude bounds (HS47, HS78,
  several CUTEst) should converge in fewer iterations after the fix; correct
  behaviour reproduces Ipopt's first iterate.

### T0.2  s_d / s_c denominators use `2n` and `m + 2n` instead of finite-bound counts
- **Where:** `src/ipm.rs:10428` (`bound_multiplier_count = 2 * n`), used in
  `compute_convergence_info_from_state`.
- **Spec:** Ipopt `IpIpoptCalculatedQuantities.cpp:3677-3699` denominator is
  `z_L.Dim() + z_U.Dim() + v_L.Dim() + v_U.Dim()` — count of *finite* bounds.
- **Impact:** when many bounds are infinite, ripopt's denominator is too
  large, deflating `s_d`, `s_c`, making the scaled gate artificially strict —
  ripopt may report MaxIterations on a point Ipopt accepts.
- **Test:** an n=100 problem with only 5 finite bounds should now converge.

### T0.3  Convergence `compl_inf` omits constraint-slack complementarity
- **Where:** `compute_compl_err_at_state` (`src/ipm.rs:10072`) returns only
  variable-bound `(x − x_L)·z_L` / `(x_U − x)·z_U` complementarity.
- **Spec:** Ipopt's `curr_complementarity` also includes the
  `(s − d_L)·v_L − μe` and `(d_U − s)·v_U − μe` blocks for slacked
  constraints.
- **Impact:** inequality-constrained problems can pass the compl test even
  while the constraint-slack residuals are far from μ. There is already a
  `complementarity_error_full` in `convergence.rs:297` that is not wired in.
- **Fix:** make `compute_convergence_info_from_state` use a compl that
  includes `max(y,0)·(g − g_L)` and `max(−y,0)·(g_U − g)` blocks (ripopt has a
  combined-y representation, so the v_L/v_U contributions enter through
  positive/negative-part splits of y).

### T0.4  `dual_inf_unscaled` is identical to scaled
- **Where:** `src/ipm.rs:10421` literally `dual_inf_unscaled: dual_inf`.
- **Spec:** Ipopt computes `unscaled_curr_dual_infeasibility(NORM_MAX)`
  separately when `obj_scaling ≠ 1`.
- **Fix:** when gradient-based obj-scaling is active (`obj_scaling != 1.0`),
  recompute the dual residual without scaling. Otherwise leave equal.

### T0.5  `acceptable_obj_change_tol` is registered but unused
- **Where:** `src/options.rs` declares it; `convergence.rs:67-88`
  (`meets_acceptable_thresholds`) never reads `last_obj`.
- **Fix:** thread `last_obj` through the convergence checker; add the
  `|curr − last|/max(1,|curr|) ≤ acceptable_obj_change_tol` gate per
  `IpOptErrorConvCheck.cpp:328-330`.

### T0.6  `diverging_iterates_tol` tested on `|f|`, not `‖x‖_∞`
- **Where:** `src/convergence.rs:138` `info.objective.abs() > 1e50`.
- **Spec:** `IpOptErrorConvCheck.cpp:255` tests `IpData().curr()->x()->Amax() >
  diverging_iterates_tol_` (default 1e20).
- **Fix:** change the test; consider a distinct `SolveStatus::DivergingIterates`
  separate from `Unbounded` (currently conflated at `ipm.rs:6226`).

### T0.7  Filter resets `θ_max` / `θ_min` on every μ change
- **Where:** `src/ipm.rs:6281-6285`.
- **Spec:** Ipopt `IpFilterLSAcceptor.cpp:325-339` sets these **once** at
  initialization from the initial constraint violation; they are never reset.
- **Bug:** repeatedly enlarging `θ_max` defeats the filter's monotonic envelope
  — late iterates can re-enter regions that earlier filter entries already
  rejected.
- **Test:** problems where ripopt cycles around the filter should now make
  monotonic progress (CONCON-class).

### T0.8  Bound multiplier reset after restoration uses `μ/slack`, not Newton+FTB
- **Where:** `src/ipm.rs:8846, 8853`.
- **Spec:** Ipopt's `RestoIpoptNLP::finalize_solution` returns the resto-NLP's
  z, which is then projected via `kappa_sigma` and standard FTB; it does not
  recompute z = μ/slack from scratch.
- **Fix:** carry the resto-phase bound multipliers back into the main IPM and
  apply only the standard `kappa_sigma` clamp; do **not** overwrite with
  `μ/slack`.

### T0.9  Filter fully reset on resto success
- **Where:** `src/ipm.rs:8972`.
- **Spec:** Ipopt does **not** reset the filter after restoration; the new
  iterate must still be acceptable to the existing filter (with the entry
  added at the resto-trigger iterate).
- **Fix:** remove the filter clear; verify filter acceptability on
  resto-return; if not acceptable, treat as resto failure.

### T0.10  η frozen at restoration entry
- **Where:** `src/restoration_nlp.rs:56`.
- **Spec:** Ipopt's resto NLP reads η dynamically from the current outer-μ via
  `eta = μ * eta_phi_factor` (a function of μ_outer); freezing it makes the
  resto-NLP non-Markov in μ.
- **Fix:** store a reference / closure for μ_outer and recompute η on each
  objective evaluation.

### T0.11  KKT synthetic z_sl ignores `_v_l`, `_v_u`
- **Where:** `src/kkt.rs:185-191, 205-211`.
- **Spec:** the slack-block multipliers v_L, v_U are first-class Lagrange
  multipliers in Ipopt's KKT — synthetic clamps `max(−y, μ/s)` lose
  information that the v_L/v_U pair already encode.
- **Fix:** propagate the actual v_L, v_U (or, since ripopt doesn't have
  separate `s` variables, the equivalent positive/negative-y parts) into the
  KKT diagonal blocks.

### T0.12  Iterative refinement runs on the wrong (condensed) system
- **Where:** `src/kkt.rs:763-799`, with `use_ic_refinement = false`
  hard-wired.
- **Spec:** Ipopt refines the **augmented** system, not the condensed one;
  `IpPDFullSpaceSolver.cpp` runs IR with the full (W + Σ) block.
- **Fix:** plumb refinement to operate on the augmented residual; expose
  `use_ic_refinement` as a real option (default true, matching Ipopt's
  `iterative_refinement_steps_required = 1`).

### T0.13  Equality δ_c always-on
- **Where:** `src/kkt.rs:248-256`.
- **Spec:** Ipopt only adds `δ_c` when the augmented system is detected
  *singular* (PerturbForSingularity), not on every solve
  (PerturbForWrongInertia handles inertia).
- **Fix:** separate the two perturbation modes; make `δ_c` conditional on
  singularity detection, not unconditional.

### T0.14  `pretend_singular` "only once" rule missing
- **Where:** `src/kkt.rs` (PD perturbation handler).
- **Spec:** Ipopt's `IpPDPerturbationHandler` allows the
  pretend-singular trick at most once per outer iteration.
- **Fix:** track a per-outer-iter flag; refuse a second pretend-singular pass.

## 3. Tier 2 — heuristic retirement (the user's primary concern)

Agent 5 inventoried 10 fallback paths in the dispatch web. These compound on
hard problems: a singular-KKT iterate triggers slack rewrite, then NE-to-LS
reformulation, then SQP fallback, then AL fallback — each invoked on the same
root-cause symptom, masking the real failure mode.

### 3.1  Remove (no Ipopt analog, no measured evidence of value)

| heuristic | location | rationale |
|---|---|---|
| ~~`try_ne_to_ls_reformulation`~~ (T2.13, removed) | dispatch | square-system reformulation has no Ipopt counterpart and conflicts with restoration. Removed entire NE-to-LS infrastructure (~688 lines). |
| ~~`try_sqp_fallback`~~ (T2.17, removed) | dispatch | a separate SQP solver is not Ipopt's recovery — restoration is. Removed `src/sqp.rs` module + `enable_sqp_fallback` option (~485 lines). |
| ~~`try_al_fallback`~~ (T2.14, removed) | dispatch | augmented-Lagrangian is not in Ipopt's spec. Removed entire `augmented_lagrangian` module + tests + `enable_al_fallback` option (~750 lines). |
| ~~`try_plain_ipm_retry`~~ (T2.15, already absent) | dispatch | retrying the same IPM is not Ipopt's strategy. Dispatch path was deleted in earlier work; nothing to remove. |
| ~~`try_slow_optimal_slack_fallback`~~ (T2.16, already absent) | dispatch | special-case slack adjustment without Ipopt analog. Dispatch path was deleted in earlier work; nothing to remove. |
| ~~`apply_late_optimality_promotion`~~ (T2.12, removed) | convergence | promoting a non-optimal point to Optimal is a CLAUDE.md "honesty" violation. **HS regression: 118 → 113 Optimal + 2 Acceptable.** Per CLAUDE.md alignment-work exception, regression accepted; root cause is convergence/scaling under-acceptance (deferred). Re-checked through T2.19 (`benchmarks/hs/hs_ripopt_results_v0.8.0-dev_T2.19.json`): cumulative state unchanged at 113/120 Optimal + 2 Acceptable; new failure modes are HS43 (compl=0.032 — bound-mult), HS214 (KKT residuals all 0 yet flagged NumericalError — convergence-test edge), HS223 (compl=1.0). |
| 3× promotion paths in convergence | `convergence.rs` (iterate averaging, active-set polish, multiplier snap) | none have an Ipopt counterpart |

**Deletion order** must respect dependencies. After T0.* lands and Tier 1
defaults match Ipopt, run the full CUTEst regression. Then *one at a time*
remove a heuristic, re-run regression, decide commit-or-revert per CLAUDE.md.
A heuristic that the regression shows is load-bearing on real problems gets
documented (not silently kept).

### 3.2  Keep but re-anchor

- `fix_inequality_mult_signs` (`ipm.rs:9810`): needed because ripopt uses a
  combined-y representation; document as a deliberate ripopt-vs-Ipopt
  representation choice.
- `early_stall_timeout`, `proactive_infeasibility_detection`:
  keep, document in the spec as ripopt-only exit shortcuts that diverge from
  Ipopt; ensure they cannot mask a successful Ipopt-style termination.
- ~~`stall_iter_limit`~~ (T2.27, default flipped 30 → 0): **retired** as the
  default; the path was masking real failures (ENGVAL2NE / PFIT4 were being
  classified Optimal at dual=3e8 / 5e12). CUTEst delta: **517 → 525 Optimal
  (+8), 532 → 543 "solved" (+11)**. Spec §14.8 documents the audit. Option
  retained as opt-in escape hatch; dead-code gate behind `stall_iter_limit > 0`.
- `try_conservative_ipm_retry` (`ipm.rs:2432`): **moved here from §3.1**
  after T2.4 deletion attempt. The HS regression suite showed it is
  load-bearing: HS TP106 (NumericalError, obj 6907 vs optimum 7049) and
  HS TP113 (NumericalError, obj 29.7 vs optimum 24.3) both depend on
  this retry. Underlying issue suspected in our Gondzio MCC / Mehrotra
  PC implementation — these problems fail with advanced corrections on
  but converge with them off. Keep until Gondzio/Mehrotra correctness
  work is a separate item.

### 3.3  Reimplement to Ipopt formula

- ~~`compute_loqo_mu` extra floors~~ (T2.28, removed): dropped the
  `monotone_floor = min(linear*mu, mu^super)` and the `loqo_mu.max(...)`
  clamp; `compute_loqo_mu` now mirrors `IpLoqoMuOracle::CalculateMu`
  exactly (`Max(Min(mu_max, sigma*avg_compl), mu_min)` with mu_max=1e5
  as the historical hard cap until §3.3 mu_max plumbing is wired
  through). CUTEst delta: net-zero (525 Optimal, 543 solved unchanged
  from T2.27); 2 problems flipped Timeout → MaxIterations from slightly
  faster iteration.
- ~~`mu_ks.max(state.mu)` extra clamp~~ (T2.3, removed): Free-mode
  mu_ks now uses `compute_avg_complementarity(state).min(1e3)` directly.
- ~~`compute_e_mu` (`ipm.rs`)~~ (T3.4, replaced): now `(dual_inf/s_d).max(primal_inf).max(compl_inf/s_c)` with finite-bound counts in `s_d`/`s_c` denominators, matching `IpIpoptCalculatedQuantities::curr_barrier_error`.
- ~~`compute_initial_y_with_ls` `m > 500` cutoff~~ (T2.2, removed):
  no outer cap; always run augmented LS, with a square-problem branch
  (m == n → y=0) matching `IpDefaultIterateInitializer::least_square_mults`.
- Quality-function oracle: **DONE (T2.23).** Implemented
  `compute_quality_function_mu` with affine + full-step KKT solves
  (option (b): re-factorises inside the oracle), golden-section
  minimisation over σ ∈ [1e-6, 1.0] with 8 max steps, optional
  `quality_function_centrality` reciprocal penalty, and a linearised
  Q(σ) using the `(1−α)·current` residual identity plus a balancing
  term that breaks the σ=1 degeneracy. Falls back to `compute_loqo_mu`
  on factorisation failure. Tech-debt versus Ipopt 3.14: Q is
  linearised (Ipopt evaluates the nonlinear residual at the trial
  point, requiring problem access in the oracle), and σ_max is capped
  at 1.0 because the linearised Q has spurious flat regions near σ=1.
  See spec §14.2 for the deviation rationale.
- **T2.24 — Magic step (spec §5.3, `IpBacktrackingLineSearch.cpp:1013-1111`)**:
  done. Added `magic_step: bool` (default `true`, mirrors Ipopt's
  `magic_steps`); implemented `compute_magic_step_delta` (Ipopt-faithful
  per-component formula: lower-bound entries take `max(0, d - s)`,
  upper-bound entries `min(0, d - s)`, doubly-bounded entries suppress
  the step when it would worsen the centering measure
  `|d_L + d_U - 2 s|`); added `apply_magic_step` and wired it into the
  post-step path in `run_main_ipm_loop` between the line-search and
  `reset_slack_multipliers`. **Architectural note**: ripopt uses an
  implicit-slack formulation, so it has no explicit slack vector `s`
  distinct from `x`; with no slack to adjust, `apply_magic_step` is a
  no-op on the standard solve path. The flag and helper are retained
  for spec compliance and as the wiring point for future
  explicit-slack paths (`slack_formulation.rs`). Unit tests cover the
  closed-form delta for lower-only / upper-only / double-bounded
  (suppress) / unbounded entries plus the no-op invariant on x, y,
  z_l, z_u, v_l, v_u in implicit-slack mode.

## 4. Tier 3 — filter / line-search fidelity

- Watchdog trial-iterate filter check: ripopt bypasses the filter when
  `alpha == alpha_primal_max` (per Agent 2). Ipopt does not bypass.
  **Fix:** always check the filter on a watchdog trial.
  **Done (T2.21):** `run_line_search_loop` now runs
  `filter.check_acceptability` on the watchdog full step and only accepts
  if the filter passes. Filter augmentation is still skipped on a
  watchdog accept, preserving Ipopt's "transient infeasibility tolerance"
  property (`IpBacktrackingLineSearch.cpp::DoBacktrackingLineSearch`).
  Locked by `test_watchdog_accept_does_not_augment_filter` in
  `src/ipm.rs::tests`.
- Tiny-step handling: ripopt forces a μ decrease on tiny step. Ipopt returns
  `STOP_AT_TINY_STEP`. **Fix:** bubble up the status.
  **Done (T2.21):** added `SolveStatus::StopAtTinyStep` mirroring
  Ipopt's `Search_Direction_Becomes_Too_Small` return code; rewrote
  `detect_tiny_step` to only set the `tiny_step` flag and bump the
  consecutive-tiny counter (no μ mutation, no filter reset). The main
  IPM loop now exits with `make_result(state, SolveStatus::StopAtTinyStep)`
  after two consecutive tiny iterations.
  `apply_late_optimality_promotion` was extended to cover the new
  status so genuinely-converged tiny-step exits still get promoted to
  `Optimal` / `Acceptable` when the KKT gates are met. Locked by
  `test_detect_tiny_step_no_mu_mutation`,
  `test_detect_tiny_step_clears_when_step_grows`, and
  `test_stop_at_tiny_step_status_roundtrip` in `src/ipm.rs::tests`.

2. Run full HS + CUTEst regression after T0 complete; save
   `benchmarks/cutest/results_v0.8.0-after-T0.json`.

4. Land Tier 4 (resto) and Tier 3 (filter) fidelity fixes.
5. Land Tier 2.3 (re-anchored heuristics) and verify metrics improve.
6. Begin Tier 2.1 deletions, one heuristic per commit, regression each time.
7. Update `IPOPT_ALGORITHM_SPEC.md` per §6.

A regression on this branch is the merge gate. CLAUDE.md "honesty in
benchmarks" means we count only `Optimal`, never `NumericalError`, never
`Acceptable`. Net change in Optimal-rate from the v0.8.0-dev baseline is the
verdict.

### 9.C  Suggested execution order

The next 6 commits should be the trivial Track A wins, in order — each
one isolated, narrow regression, commit-or-revert per CLAUDE.md:

1. **T3.6** (verify s_d/s_c counts) and **T3.7** (verify kappa_sigma scope) and **T3.8** (verify free-mode τ source) — three audits that may need no fix but might surface silent bugs. Do these *first* because they may invalidate or amplify everything else.

3. **T3.2** plumb `mu_max_fact`.
4. **T3.4** `obj_max_inc` guard, then **T3.3** `theta_max`, then **T3.5** full `alpha_min` (filter-LS triplet).

Track A items have settled the regression baseline; otherwise we conflate
"removed a heuristic that was masking a real bug" with "removed a

introduced wrong elsewhere".
