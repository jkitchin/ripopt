# ripopt Benchmark Suites

This directory collects the benchmark suites ripopt runs against for
performance and correctness tracking. Each subdirectory is self-contained
with its own README, input data, and results.

Run `make help` from this directory for the list of targets, or
`make benchmark` for the full sweep.

## Suites

| Suite       | Path                      | Description                                                                                               |
|-------------|---------------------------|-----------------------------------------------------------------------------------------------------------|
| HS          | `benchmarks/hs/`          | Hock-Schittkowski test set (120 classic NLPs)                                                             |
| CUTEst      | `benchmarks/cutest/`      | CUTEst SIF problems (727 NLPs covering bounds/equality/inequality)                                        |
| Electrolyte | `benchmarks/electrolyte/` | Electrolyte thermodynamic equilibrium (chemistry)                                                         |
| Grid        | `benchmarks/grid/`        | AC optimal power flow on electrical grids (IEEE/PGLib MATPOWER cases)                                     |
| CHO         | `benchmarks/cho/`         | CHO parameter estimation (kinetics)                                                                       |
| Large-scale | `benchmarks/large_scale/` | Synthetic NLPs, up to 100K variables                                                                      |
| Gas         | `benchmarks/gas/`         | 4 gas pipeline NLPs from PDE discretization, n=204 to n=21058, source: Sakshi21299/gas_networks issue #12 |
| Water       | `benchmarks/water/`       | 6 water distribution network design NLPs from MINLPLib (Drud & Rosenborg / Bragalli family)               |
| Preprocessing | `benchmarks/preprocessing/` | AMPL `.nl` comparison of `enable_preprocessing=yes` vs `enable_preprocessing=no` on in-repo instances |

The HS, CUTEst, electrolyte, grid, and large-scale suites feed into the
composite `benchmarks/BENCHMARK_REPORT.md` generated by
`benchmarks/benchmark_report.py`. The CHO suite is loaded opportunistically
when `benchmarks/cho/cho_results.json` exists. The gas and water suites are
standalone: problems are solved via the AMPL `.nl` interface with results
written to per-problem `.sol` files, no JSON aggregation, and they do not
feed the composite report — see `benchmarks/{gas,water}/README.md` for
details.

The preprocessing suite is a focused A/B benchmark for the auxiliary
preprocessing implementation. It writes compact JSON and Markdown reports for
pull-request discussion instead of feeding the composite Ipopt comparison.
